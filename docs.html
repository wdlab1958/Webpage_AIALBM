Memory System

3-layer hierarchical memory architecture with automatic consolidation, vector search, and semantic retrieval.

STM

Short-Term Memory (STM)

Immediate context buffer holding current conversation state. Fast access, limited capacity, auto-expires after session.

MTM

Medium-Term Memory (MTM)

Working memory for active tasks. Stores patterns, preferences, and recent interactions for cross-session continuity.

LTM

Long-Term Memory (LTM)

Persistent knowledge store. Consolidated patterns, learned behaviors, and domain expertise preserved indefinitely.

Memory Consolidation Engine

Automated pattern merging and compression. Redundant memories are deduplicated, and related memories are linked in the knowledge graph.

Vector Search

SentenceTransformer (all-MiniLM-L6-v2) embeddings indexed in FAISS and Qdrant for millisecond semantic retrieval.

Hybrid Search

Combines dense vector similarity with BM25 keyword matching via Reciprocal Rank Fusion for maximum recall.

Knowledge Graph

Entity-relation graph representing complex knowledge relationships. Supports multi-hop reasoning and contextual enrichment.

WebSocket Stream

Real-time bidirectional communication for streaming AI responses, typing indicators, system event broadcasts, and connection management.

Token-by-Token Streaming

AI responses streamed in real-time, token by token. Provides immediate feedback with progressive rendering as the model generates output.

Typing Indicators

Real-time typing status awareness. Both user and AI typing states are broadcast to all connected clients in the conversation.

System Event Broadcasts

Push notifications for system events: agent status changes, memory consolidation completion, training milestones, and health alerts.

Connection Management

Automatic reconnection with exponential backoff. Heartbeat ping/pong for connection health. Multi-client support per conversation.

WebSocket Protocol

Connection Endpoint

ws://localhost:8003/api/v1/ws/chat/{conversation_id}

Message Types

messageSend chat message to AI agent
typingBroadcast typing indicator
stream_startAI begins generating response
stream_tokenIndividual token in streaming response
stream_endAI response generation complete
systemSystem event notification
errorError notification

WebSocket Client Example

const ws = new WebSocket(
'ws://localhost:8003/api/v1/ws/chat/conv_123'
);
ws.onmessage = (event) => {
const data = JSON.parse(event.data);
if (data.type === 'stream_token') {
process.stdout.write(data.content);
}
};
ws.send(JSON.stringify({
type: 'message',
content: 'Hello AI',
agent_type: 'claude'
}));
